# Git Branch 是什麼？

## 為什麼需要分支？

想像你在開發一個專案：

> 主線 `main` 已經上線，用戶正在使用。  
> 你想開發新功能，但不想把未完成的程式碼直接改到主線上。

**分支**讓你可以**平行開發**，互不干擾。

```
main ──────●──────●──────●──────●──────●───────
              \                    /
feature/xxx    ●────●────●───────
```

---

## 核心觀念

| 概念 | 說明 |
|------|------|
| **主分支** | 通常叫 `main` 或 `master`，代表穩定可發布的版本 |
| **功能分支** | 從主分支分出，用來開發新功能 |
| **修復分支** | 用來修 bug，修完後合併回主分支 |

> 💡 分支本質上就是**指向某個 commit 的指標**，建立分支幾乎不佔空間。

---

## 常見使用場景

- 開發新功能（不影響主線）
- 多人協作（每人負責不同分支）
- 修復 bug（緊急修復用 hotfix 分支）
- 實驗性改動（不確定要不要，先開分支試試）
