# Context 上下文指令

## @ 符號引用

在 Chat 輸入框輸入 `@` 可引用各種上下文：

| 引用類型 | 用法 | 說明 |
|----------|------|------|
| `@檔案` | `@src/App.tsx` | 引用特定檔案 |
| `@資料夾` | `@src/components` | 引用整個目錄 |
| `@程式碼` | `@函數名` / `@類名` | 引用程式碼符號 |
| `@文檔` | `@docs` | 引用已索引的外部文檔 |
| `@Git` | `@Git` | 引用 Git 提交或差異 |
| `@終端` | `@終端` | 引用終端輸出 |
| `@Cursor Rules` | `@Cursor Rules` | 引用規則檔 |
| `@Web` | `@Web` | 即時搜尋網路 |

---

## Codebase 語義搜尋

- Cursor 會自動索引整個專案
- 輸入 `@Codebase` 可觸發**語義搜尋**，依意思找相關程式碼
- 適合：不確定檔名時，用描述性關鍵字搜尋

---

## 精確引用技巧

| 技巧 | 範例 |
|------|------|
| 引用特定行 | `@App.tsx:42` 引用第 42 行附近 |
| 避免膨脹 | 優先引用單一檔案，少用整個資料夾 |
| 新話題 | 用 `Cmd+N` 開新對話，避免舊上下文干擾 |

---

## 管理 Context 的原則

1. **精準**：只引用與問題相關的檔案
2. **適量**：過多引用會佔用 Token、影響回答品質
3. **更新**：修改程式碼後，必要時重新引用
