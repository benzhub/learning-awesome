# Cursor Codebase Indexing — 用語義理解你的程式碼庫

## 這是什麼
Codebase Indexing 是 Cursor 的語義索引能力。它會把專案檔案轉成可檢索的語義向量，讓你用「描述問題」而不是「記住檔名」就能找到相關程式碼。

和傳統全文搜尋差異：全文搜尋看字面關鍵字；Codebase Indexing 看語意相近度，所以你問「登入流程在哪」時，也可能找到 `auth`、`session`、`token` 相關實作。

## 適用場景
- ✅ 你剛接手陌生專案，不知道從哪個檔案開始看
- ✅ 你要追功能路徑（例如：下單、付款、登入）但缺少明確函式名
- ✅ 你想快速定位相似實作（例如：另一個 API 的錯誤處理）
- ❌ 不適合：你已經知道精確檔名與行號（此時直接 `@檔案:行` 更快）

## 快速啟用
在 Cursor 的 Indexing & Docs 確認 Codebase Indexing 已完成（例如顯示 100%），再把下列內容直接貼到 Chat 測試：

```text
@Codebase
請告訴我「使用者登入」的完整流程：
1) 入口點在哪裡
2) 驗證邏輯在哪裡
3) 成功後資料寫入哪裡
請列出涉及的檔案路徑與關鍵函式。
```

> 💡 重點不是關鍵字多，而是需求描述清楚。先給「任務目標」，再給「你要的輸出格式」。
