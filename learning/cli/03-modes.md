# 03 — 三種模式：Agent / Plan / Ask

## 一句話區分

| 模式 | 一句話 | 會不會改你的檔案？ |
|------|--------|--------------------|
| **Agent** | 幫我「做」一件事 | ✅ 會 |
| **Plan** | 先「規劃」怎麼做，再決定要不要執行 | 先不動，你同意才動 |
| **Ask** | 只「問」問題、查資料 | ❌ 不會 |

---

## Agent 模式（預設）

**用途**：請 AI 直接改 code、加功能、重構。

```bash
cursor agent "把登入改成用 JWT"
```

- 會讀檔案、改檔案、可能執行指令
- 每一步可以選擇接受或拒絕

---

## Plan 模式

**用途**：不確定怎麼做時，先看 AI 的計畫再決定。

```bash
cursor agent --plan "重構整個 API 層"
```

- AI 會先問你幾個問題、提出做法
- 你同意後才真的去改

**什麼時候用**：大改動、想先看清楚再動手時。

---

## Ask 模式（唯讀）

**用途**：只想問「這段 code 在幹嘛」「有沒有更好的寫法」，**不要改任何檔案**。

```bash
cursor ask "src/auth.ts 的邏輯是什麼？"
```

- 只會讀取、分析、回答
- 不會寫入檔案，適合在 CI 或不敢亂動的環境用

---

## 在對話裡切換模式

進入 `cursor agent` 對話後，可以用 **slash 指令** 切換：

- `/plan` → 切到 Plan 模式
- `/ask` → 切到 Ask 模式（唯讀）

---

## 小結

- **Agent**：做事情、會改檔案
- **Plan**：先規劃、你同意才執行
- **Ask**：只問不改，唯讀
